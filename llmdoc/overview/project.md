# 项目概述

## 基本信息

| 属性 | 值 |
|------|-----|
| 产品名称 | 不远不近 (where-to-meet) |
| Slogan | 不远不近，刚刚好 |
| 产品类型 | 微信小程序 |
| 当前版本 | V1.0 MVP |
| 覆盖范围 | 北京（MVP 限定） |

## 产品定位

帮助多人（2-8人）快速找到「对所有人距离都公平」的聚会地点。用户输入各自位置，选择活动类型（聚餐、KTV 等），系统基于综合评分算法推荐最优地点。

## 核心用户场景

| 场景 | 描述 |
|------|------|
| 朋友聚餐 | 3 个朋友分别在望京、国贸、中关村，想找大家都方便的餐厅 |
| 同事团建 | 6 个同事从不同住处出发，找距离均衡的 KTV |
| 情侣约会 | 两人一个在东边一个在西边，找中间点附近的咖啡厅 |
| 临时约局 | 微信群里有人发起聚会，快速确定地点 |

## 核心价值

- **省时** — 不用在群里反复讨论"去哪儿"，一键获得推荐
- **公平** — 推荐结果兼顾所有人的距离，不让任何人跑太远
- **靠谱** — 结合地点评分和距离的综合排序，推荐质量可信赖
